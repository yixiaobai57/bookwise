# BookWise 设计文档

> 根据英语词汇量 AI 识别书籍库内词汇覆盖率，自动为用户推荐合适书籍的网站。

**日期**：2026-06-24  
**状态**：设计完成，待实现

---

## 1. 项目概述

BookWise 帮助中国英语学习者找到词汇覆盖率达 80% 的全英文书籍。用户通过多种方式输入词汇量，系统结合词形还原匹配与 AI 语义分析计算每本书的覆盖率，自动推荐最合适的书籍。

**目标用户**：中国英语学习者  
**技术栈**：Next.js 14 + Vercel Serverless + Tailwind CSS + Framer Motion  
**部署**：GitHub 仓库连接 Vercel，自动 CI/CD

---

## 2. 功能需求

### 2.1 词汇量输入（四种方式组合）

#### A. 在线词汇测试（核心）
- 自适应测试算法，从 COCA 词频表按频率分层抽样
- 每轮展示 1 个单词，用户选择"认识"或"不认识"
- 根据回答动态调整难度（连续答对提升词频等级，连续答错降低）
- 测试 30-50 个词后，用贝叶斯推算总词汇量
- 词库覆盖 COCA 前 20,000 词，带中文释义

#### B. 直接输入词汇量
- 用户输入数字，系统直接采用
- 范围校验：1,000 - 30,000

#### C. 考试成绩映射
- 用户选择考试类型 + 输入分数
- 映射表：

| 考试 | 分数范围 | 词汇量映射 |
|------|----------|------------|
| 四级 | 425-550 | 4,000-5,500 |
| 四级 | 550-710 | 5,500-7,000 |
| 六级 | 425-550 | 5,500-7,000 |
| 六级 | 550-710 | 7,000-9,000 |
| 考研 | 50-70 | 5,500-7,500 |
| 考研 | 70-100 | 7,500-10,000 |
| 托福 | 80-100 | 8,000-12,000 |
| 雅思 | 6.0-7.5 | 7,000-10,000 |

#### D. 导入单词表
- 用户粘贴文本或上传文件（TXT/CSV）
- 系统提取英文单词 → 去重 → 与词库匹配 → 输出已知词汇量

**存储格式**（localStorage）：
```json
{
  "vocabularySize": 8500,
  "testMethod": "adaptive",
  "testedAt": "2026-06-24T10:00:00Z",
  "knownWords": ["abandon", "ability", ...]
}
```

### 2.2 书籍数据来源

#### 精选书库（预处理 JSON）
- 80-120 本经典英文书，预处理为 JSON 存放在仓库
- 元数据结构：
```json
{
  "id": "pride-and-prejudice",
  "title": "Pride and Prejudice",
  "titleCN": "傲慢与偏见",
  "author": "Jane Austen",
  "category": "小说",
  "difficulty": "六级",
  "wordCount": 122000,
  "uniqueWords": ["abandon", "ability", ...],
  "wordFrequencyDistribution": {
    "top3000": 85,
    "3001-6000": 10,
    "6001-10000": 3,
    "10000+": 2
  }
}
```
- 分类标签：小说 / 科幻 / 历史 / 科普 / 哲学 / 传记 / 教材
- 难度标签：四级 / 六级 / 考研 / 托福 / 雅思

#### Gutenberg 文本
- 通过 Gutenberg API 实时拉取全文
- 拉取后在 Serverless Function 中做词形还原处理，结果缓存

### 2.3 覆盖率计算

**第一步：词形还原匹配**
1. 将书籍全文做词形还原（lemmatization），统计唯一词表
2. 与用户已知词表（或词汇量对应的词频等级）对比
3. `覆盖率 = 已知词数 / 书籍唯一词总数 × 100%`

**第二步：AI 语义分析**
- 对书籍前 2000 词调用 Hugging Face 免费模型
- 分析维度：句子复杂度、隐喻密度、文化背景依赖度
- 输出难度修正系数（0.9-1.1）

**最终覆盖率** = 词形还原覆盖率 × AI 难度修正系数

### 2.4 推荐算法

```
输入：用户词汇量 (8500)、目标覆盖率 (80%)
遍历书库 → 计算每本书的覆盖率
筛选：覆盖率 ≥ 80% 的书
排序：覆盖率从高到低
输出：推荐列表 + 每本书的覆盖率百分比
```

### 2.5 MVP 核心功能

1. **词汇量测试** — 用户完成测试后得到词汇量数字
2. **书籍推荐列表** — 根据词汇量自动筛选 80% 覆盖率的书
3. **书籍详情页** — 展示覆盖率、难度分析、简介

## 3. 整体架构

```
┌─────────────────────────────────────────────────┐
│                    浏览器端                       │
│  ┌───────────┐  ┌───────────┐  ┌──────────────┐ │
│  │ 词汇测试   │  │ 书籍推荐   │  │ 书籍详情页   │ │
│  └─────┬─────┘  └─────┬─────┘  └──────┬───────┘ │
│        └──────────────┼───────────────┘          │
│              ┌────────▼────────┐                 │
│              │  localStorage   │                 │
│              │  (词汇量/历史)   │                 │
│              └─────────────────┘                 │
└─────────────────────────────────────────────────┘
                       │ API 调用
                       ▼
┌─────────────────────────────────────────────────┐
│              Vercel Serverless Functions          │
│  ┌───────────────┐  ┌──────────────────────────┐│
│  │ /api/test     │  │ /api/recommend            ││
│  └───────────────┘  └──────────────────────────┘│
│  ┌───────────────┐  ┌──────────────────────────┐│
│  │ /api/gutenberg│  │ /api/analyze              ││
│  └───────────────┘  └──────────────────────────┘│
└─────────────────────────────────────────────────┘
                       │
          ┌────────────┼────────────┐
          ▼            ▼            ▼
   ┌────────────┐ ┌─────────┐ ┌──────────┐
   │ Gutenberg  │ │精选书库  │ │ Hugging  │
   │ API        │ │(JSON)   │ │ Face API │
   └────────────┘ └─────────┘ └──────────┘
```

**核心数据流**：
1. 用户完成词汇测试 → 得到词汇量数字 → 存入 localStorage
2. 进入推荐页 → 系统从精选书库 + Gutenberg 拉取书籍 → 用词汇量计算覆盖率 → 筛选出 ≥80% 的书展示
3. 用户点击书籍详情 → 展示 AI 语义分析结果 + 详细覆盖率数据

## 4. 页面与 UI 设计

### 4.1 视觉风格

对标 open-design.ai 的现代渐变风格：
- 大字标题，关键词用渐变色高亮
- 极简留白，内容不拥挤
- 柔和蓝紫渐变，不刺眼
- 微动效：fade-up 进入、hover 上浮
- 白色卡片 + 轻微阴影 + 圆角

### 4.2 配色方案

| 用途 | 色值 |
|------|------|
| 主渐变（按钮/强调文字） | #6366f1 → #a855f7（靛蓝→紫） |
| 背景 | #fafafa |
| 卡片背景 | #ffffff |
| 主文字 | #1a1a2e |
| 次要文字 | #6b7280 |
| 覆盖率高 | #22c55e |
| 覆盖率中 | #eab308 |
| 覆盖率低 | #ef4444 |

### 4.3 页面路由

```
/                    → 首页
/test                → 词汇量测试
/recommend           → 书籍推荐列表
/book/:id            → 书籍详情页
```

### 4.4 各页面设计

#### 首页 `/`
- Hero：超大字号标题"找到适合你词汇量的英文书"，"词汇量"用渐变色高亮
- 副标题："AI 智能分析，每次推荐 80% 词汇覆盖率的书籍"
- CTA 按钮"开始测试 →"，渐变背景，hover 上浮
- 滚动后：三个功能卡片 fade-up 逐个进入

#### 测试页 `/test`
- 大字单词居中展示 + "认识"（绿）/ "不认识"（灰）按钮
- 顶部细进度条（渐变色）
- 结果页：词汇量超大字号 + 渐变色 + 对标等级

#### 推荐页 `/recommend`
- 顶部筛选标签栏（圆角胶囊按钮，选中态渐变背景）
- 书籍卡片网格：封面 + 书名 + 作者 + 覆盖率圆形进度条
- 卡片 hover：上浮 + 阴影加深

#### 详情页 `/book/:id`
- 大标题区：书名（渐变高亮）+ 作者 + 分类标签
- 覆盖率展示：圆形大进度条 + "你的词汇覆盖 87%" 大字
- AI 分析区域：卡片式难度分析摘要

### 4.5 动效规范

- 页面切换：淡入 + 轻微上移（300ms ease-out）
- 卡片进入：Intersection Observer 触发 fade-up，逐个延迟 100ms
- hover：translateY(-4px) + box-shadow 加深（200ms）
- 按钮 hover：scale(1.02) + 阴影扩散
- 加载状态：骨架屏（shimmer 动画）

### 5.3 API 设计

#### `POST /api/test`
- 输入：`[{ word: "abandon", known: true }, ...]`
- 输出：`{ vocabularySize: 8500, confidence: 0.92 }`

#### `POST /api/recommend`
- 输入：`{ vocabularySize: 8500, targetCoverage: 0.8, category?, difficulty? }`
- 输出：`[{ id, title, titleCN, author, coverage, category, difficulty }]`

#### `GET /api/gutenberg?id=1342`
- 输出：`{ text, uniqueWords, wordCount }`

#### `POST /api/analyze`
- 输入：`{ bookId, textSnippet }`
- 输出：`{ difficultyScore: 0.85, analysis: "...", correctionFactor: 0.95 }`
